Troubleshooting

Almost everything self-heals or can be fixed by asking an assistant. Here are the few things that sometimes need you.

🤖

Fastest fix: open claude and paste whatever red text you saw, then say “please fix this.” The sections below are for when you'd rather do it yourself.

“claude can't be opened — unidentified developer”

macOS Gatekeeper being cautious. The app is signed by Anthropic and safe. Clear the flag:

xattr -d com.apple.quarantine ~/.local/bin/claude

Two different claude versions

If claude behaves oddly, you might have both the native install and an old npm copy. See them all, then remove the npm one:

which -a claude
npm uninstall -g @anthropic-ai/claude-code

The one you want lives at ~/.local/bin/claude and updates itself.

“Refusing to load formula … from untrusted tap”

Homebrew 6 asks you to approve third-party “taps” before installing from them. If an install is refused, trust that tap and retry. For example, for the VM test tool:

brew trust cirruslabs/cli
brew install cirruslabs/cli/tart

The setup scripts do this automatically; you'd only see it if you install something extra by hand.

ngrok says it needs an auth token

ngrok (for sharing a local site over a temporary public link) needs a free token. Get one at dashboard.ngrok.com, then:

ngrok config add-authtoken <your-token>

Xcode license / “first launch” (mobile setup)

After Xcode installs, it needs a one-time license acceptance and setup. Run:

sudo xcodebuild -license accept
xcodebuild -runFirstLaunch

For Android, open Android Studio once and complete its setup wizard — that installs the Android SDK.

The GitHub assistant tool isn't working

It needs GitHub to be signed in. Check, and log in if needed:

gh auth status
gh auth login

Then re-run the agent setup: cd ~/Developer/mac-launchpad && bash modules/05-agents.sh.

Antigravity (agy) says “command not found”

Open a fresh Terminal window first — the installer adds ~/.local/bin to your path for new sessions. If it's still missing:

ls ~/.local/bin/agy            # confirm it's installed
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

Update Antigravity any time with agy update.

Antigravity won't sign in

Run agy and finish the Google sign-in in the browser (it uses Google Chrome). If no browser opens — for example over SSH — Antigravity prints a URL; open it on any machine, approve, and you're in. Your login is stored safely in the macOS Keychain; type /logout inside agy to clear it.

🏢

Work/enterprise Google account? If your organisation requires signing in through Google Cloud instead of a personal Gmail, install the Google Cloud SDK (brew install --cask google-cloud-sdk) and run gcloud auth login first. Personal Gmail accounts don't need this — it's not installed by default.

“publish to here.now” isn't working

All three assistants can publish sites and store files through the built-in here.now skill. If it's missing, check it's installed for the assistant you're using:

ls ~/.claude/skills/here-now/SKILL.md                      # Claude Code
ls ~/.agents/skills/here-now/SKILL.md                      # Codex
ls ~/.gemini/antigravity-cli/skills/here-now/SKILL.md      # Antigravity

Re-install all three with cd ~/Developer/mac-launchpad && bash modules/05-agents.sh. Anonymous publishing needs no account. For permanent sites, custom domains, or private Drives, get a free key at here.now/dashboard and save it:

mkdir -p ~/.herenow && echo "YOUR_KEY" > ~/.herenow/credentials && chmod 600 ~/.herenow/credentials
🌍

here.now sites are public by default. Ask for a password-protected or restricted site for anything sensitive, and never publish secrets or private files to an anonymous link.

Managing the assistants' skills

Your assistants share a curated set of skills — the Superpowers engineering workflow (brainstorm → plan → test → review), plus design, document, and browser skills. To see and refresh them:

npx skills list                # skills shared across all three assistants
claude plugin list             # Claude Code's plugins, including Superpowers
cd ~/Developer/mac-launchpad && bash modules/06-skills.sh   # re-install them all

Superpowers runs as a full plugin for Claude Code (its files live under ~/.config/superpowers/ and ~/.claude/plugins/). Codex and Antigravity use the same skills in “degraded mode” — the skill files plus the shared house-rules in ~/.codex/AGENTS.md / ~/.gemini/AGENTS.md — because they don't support Claude Code's full plugin framework. They still follow the brainstorm-first workflow; they just don't have its automatic enforcement hooks.

🔁

Just installed Superpowers and Claude isn't using it? Restart claude — plugins load at the start of a session.

Tests or the browser tools aren't working

agent-browser (the assistants' live browser) downloads its own Chrome the first time. Check it's healthy or re-fetch it:

agent-browser doctor          # reports Chrome, the daemon, and providers
agent-browser install         # re-download Chrome for Testing if needed

Playwright e2e browsers are cached in ~/Library/Caches/ms-playwright, shared by every project. If a test says a browser is missing, run npx playwright install.

Maestro (mobile tests) lives in ~/.maestro/bin and needs Java 17+ (installed with the mobile setup). If maestro isn't found, open a fresh terminal or add it to your path: export PATH="$PATH:$HOME/.maestro/bin".

CI failing on GitHub? Open the repo's Actions tab, click the red run, and read the step that failed — then paste that text to an assistant and ask it to fix the test. Each run also saves the Playwright report as a downloadable artifact.

Tighten the AI's “blast radius”

By default the assistants run freely so setup never stalls. If you'd prefer they stay inside your ~/Developer folder and can't touch the rest of your Mac, change one setting.

Codex

Edit ~/.codex/config.toml and change the sandbox line to:

sandbox_mode = "workspace-write"

That limits writes to the current project folder. (The wide-open setting is danger-full-access.)

Claude Code

Edit ~/.claude/settings.json and change "defaultMode" from "bypassPermissions" to "acceptEdits" (auto-approves file edits but asks before running commands) or "default" (asks for everything).

Antigravity

Antigravity is autonomous by default here too. To run it with permission prompts for one session, type command agy (or \agy) instead of agy. Inside a session, /permissions switches between “request review,” “always proceed,” and stricter modes.

⚖️

Tighter = safer but more interruptions. For a brand-new builder, the default open setting plus Git checkpoints and Time Machine is usually the right trade.

Text looks like boxes (□) or icons are missing

An app isn't using the coding font. In VS Code/Cursor it's set for you; if an app still shows boxes, set its font to JetBrainsMono Nerd Font in that app's settings.

Start fresh / health check

Everything is safe to run again. To re-check or re-fix your whole setup:

cd ~/Developer/mac-launchpad
bash lib/doctor.sh          # green ✔ / red ✘ checklist
bash scripts/update.sh      # update everything and re-check
🪵

The full setup log is at ~/launchpad-setup.log — that's the one file to share if you ask someone for help.
