Build your first app

In about ten minutes you'll have a real web app running on your Mac — built, changed, and saved. This is the exercise Claude Code runs with you right after setup.

🤖

You can let the assistant do every step (just say “walk me through building my first app”), or follow along yourself below. Either way, you'll understand what happened.

1Make a project

In the Terminal:

mkproj hello-launchpad
claude

This creates ~/Developer/hello-launchpad, turns on save-points (Git), and starts Claude Code inside it.

2Ask for the app

Tell Claude exactly this:

Create a small Vite + React + TypeScript app here. Give it a friendly
welcome page with my name and a button that, when clicked, shows a random
encouraging message. Then start it so I can see it.

Claude will set up the project, write the code, and start a local development server — a little web server running on your Mac just for you.

🧰

Behind the scenes it uses Vite (builds and serves the app), React (builds the page), and TypeScript (catches mistakes early). You don't need to know these yet — but now you've met them.

3See it running

Claude will give you a link like http://localhost:5173. It can also open a browser itself (using its built-in Playwright tool) to confirm the page works before handing it to you. Open that link in your browser — that's your app.

🌐

localhost just means “this computer.” Only you can see it until you choose to publish.

4Change something

Ask for a tweak and watch it update instantly (no refresh needed):

Make the headline bigger and change the background to a soft gradient.

This instant-update is called hot reload. Keep asking for changes until you like it — “add a dark mode,” “use a rounded button,” “make it bounce.”

5Save a checkpoint

Lock in your progress:

Save this as a checkpoint with a clear message.

Claude makes a commit — a save point you can always return to. From now on you can fearlessly experiment; if anything breaks, say “undo the last change.”

6(Optional) Put it online

Want to share it? Ask:

Publish this so I can send someone a link.

Claude will deploy it (usually to Vercel, free to start) and give you a real URL. The first time, you'll sign in once.

🎉 You just built and ran an app

That's the whole loop, and it's the same for anything bigger: describe → build → run → change → save → share. You don't memorize commands; you describe what you want and review the result.

Next, dream up something of your own — a tool you wish existed, a game, a site for something you love — and just ask for it.
