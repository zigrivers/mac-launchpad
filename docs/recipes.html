Recipes

The hardest part isn't the tools — it's knowing what to say. Copy a prompt below, paste it to any assistant (claude, codex, or agy), and tweak the details. They're starting points; say it however feels natural.

🧭

Your assistants plan before they build (the “Superpowers” workflow), so the first reply is often a few clarifying questions. That's normal — answer them and you'll get much better results.

Start something

From an empty terminal, the fastest start is a ready-to-run template:

launchpad new

Or just describe the whole idea and let an assistant scaffold it:

Build me a one-page website for my bakery: a hero photo, a menu
section, an about blurb, and a contact form. Warm, modern, mobile-friendly.
Start from the web template, run it, and show me.
Make a habit tracker app: I add habits, check them off each day, and
see a streak counter. Save my data on my computer for now. Write tests
and show it running in my browser.

Add a feature

Run these inside an existing project (after cd-ing into it and starting an assistant).

User login

Add user accounts to this app: sign up, log in, log out, and a
“you must be logged in” guard on the main page. Use Supabase for auth.
Walk me through creating the free Supabase account and where to paste the
keys, keep them in .env.local, and add a couple of tests.

A database table

I want to store “projects” for each user (a title, a description, and a
created date). Add the database table, the code to create/read/update/
delete them, and a simple page to manage them. Explain what you set up.

Take payments

Add a paid plan with Stripe: a “Subscribe” button, Stripe Checkout, and
unlock a premium page after payment. Use Stripe TEST mode, walk me through
getting my test keys, and never commit them. Add a webhook to confirm payment.

File or image upload

Let users upload a profile photo and show it. Store the file safely,
limit it to images under 5 MB, and handle the “upload failed” case nicely.

Search, dark mode, and the small stuff

Add a search box that filters the list as I type.
Add a dark-mode toggle that remembers my choice.

Make it work on phones

Make this look great on a phone: stack the layout, bigger tap targets,
no sideways scrolling. Then open it at a phone size and show me a screenshot.

Building an actual phone app instead? Start the mobile template and run it on your phone:

launchpad new        # pick “mobile”, then: npx expo start

Make it look good

Give this a clean, modern visual design — good spacing, a nice font
pairing, a calm colour palette, and a clear visual hierarchy. Use your
design skills and show me before/after.
This feels a bit plain. Add some polish: subtle hover states, a little
motion when things appear, and a proper empty state when there's no data.

Test it & fix it

Write tests

Write tests for what we just built — unit tests for the logic and an
end-to-end test for the main flow — then run them and show me they pass.

Something broke

Something's broken. Check Sentry for the latest error, explain in plain
English what went wrong, fix it, and add a test so it can't happen again.

No Sentry set up yet? Paste any red error text you see and say “please fix this.”

Accessibility

Run an accessibility check on the main pages and fix what it finds —
labels, colour contrast, and keyboard navigation.

See your data

Add a few realistic example rows to my database so I have something to
look at, and tell me how to open Beekeeper Studio to see them.

Then open the GUI yourself any time:

open -a "Beekeeper Studio"

Keep my keys safe (1Password)

If you use 1Password, your API keys can live there instead of a plain file — and an assistant can run your app with them injected in memory, never written to disk. Optional: if you skip it, keys just live in a git-ignored .env.local as usual.

launchpad secrets set STRIPE_SECRET_KEY   # store a key (in 1Password if signed in)
launchpad secrets run -- npm run dev      # run with secrets injected, nothing on disk
launchpad secrets status                  # which mode am I in?

Not signed in to 1Password? Every command falls back to .env.local and tells you so — turn 1Password on anytime with op signin.

Put it online

Deploy this so I can share a link. Use Vercel (free to start), connect
my GitHub repo, and give me the URL. Explain what's now public.

For an instant share of a single page, image, or PDF — no account needed:

Publish this page to here.now and give me the link.
🌍

here.now links are public by default. For anything private, ask for a password-protected or restricted site, and never publish secrets.

Save, undo & get help

Save this as a checkpoint with a clear message, and push it to my
private backup on GitHub.
That last change made it worse — undo it and go back to the previous
checkpoint.

If you need to send your setup to someone for help, make a secret-free report file:

launchpad report
