Cheat sheet

Everything you need on one page. When in doubt, just ask an assistant in plain English.

Start an assistant

claude
Start Claude Code in the current folder.
codex
Start OpenAI Codex in the current folder.
agy
Start Google Antigravity in the current folder.
/exit
Leave Claude / Codex (or Ctrl+C twice). In Antigravity, type /quit.
claude mcp list
Check the assistant's connected tools (docs, browser, GitHub, files).

Inside Antigravity (agy):

/model
Switch the AI model (Gemini, Claude, GPT…).
/usage
See your remaining quota.
/logout
Sign out of Google.

Make a project

launchpad new
Start from a ready-to-run template (web, phone, or game). Sets up Git, a secret-scanning hook, and a private GitHub backup automatically.
mkproj name
Make an empty project at ~/Developer/name with the same safety net, and go in.
launchpad harden .
Add that safety net (hook + private backup) to a folder you already have.
dev
Jump to your ~/Developer folder.
cd name
Go into a folder called name. cd .. goes back up.

Everyday terminal

ll
List files here (with details and Git status).
lt
Show the folder as a tree.
cat file
Show a file's contents (nicely colored).
Ctrl‑R
Fuzzy-search your past commands. Ctrl+T finds files.
reload
Reload terminal settings after a change.

Save & undo (checkpoints)

gs
See what's changed since your last save point.
“save a checkpoint”
Ask the assistant — it commits your work with a clear message.
“undo the last change”
Ask the assistant to roll back to the previous checkpoint.
g lg
See your project's history of save points.

Run & share

“run it”
Ask the assistant to start the app and open it in your browser.
npm run dev
Start most web apps locally (the assistant usually does this for you).
“put this online”
Ask the assistant to deploy (Vercel for web) and give you a link.
“publish to here.now”
Instantly publish a page or file to a shareable link (anonymous links last 24h; public by default).
ports
See what's currently running and on which port.

Testing

npm test
Run unit / component tests (Vitest).
npx playwright test
Run end-to-end + accessibility + visual tests.
“write tests for this”
Ask an assistant to add and run tests for a feature.
maestro test flow.yaml
Run a mobile-app test flow (mobile setups).

The assistants also drive a real browser themselves (via agent-browser) to check the app, and every push to GitHub re-runs the tests automatically if you copied in the CI workflow.

Containers (Docker / OrbStack)

open -a OrbStack
Start Docker & see running containers, logs, and their files.
“add a database”
Ask an assistant — it wires up Postgres/Redis with Docker Compose.
docker compose up -d
Start a project's local services.
name.orb.local
Open a running container in your browser by name.
fly launch
Package & deploy a container app (sign in first: fly auth login).

Safety, data & backups

Private GitHub backup
Every project is pushed to a private GitHub repo — your real off-machine backup. Ask an assistant to “push my work” any time.
Secret scanning
A pre-commit hook blocks any commit that contains a key or password. Don't use git commit --no-verify.
open -a "Beekeeper Studio"
See and edit your database in a friendly window — no SQL needed.
“check Sentry and fix it”
When something breaks, an assistant reads the real error from Sentry and fixes it.
launchpad report
Make one secret-free file describing your setup to send for help (--share for a private link).
ccusage
See your Claude Code token usage and estimated cost.
launchpad spend
See today's and this month's AI spend. You'll also get a notification if spending suddenly spikes.
launchpad doctor --fix
Health-check and auto-repair anything red (re-installs what's missing).
launchpad nudge off
Turn off the gentle "you have unsaved work" reminder (on to bring it back).
launchpad secrets set NAME
Store an API key (in 1Password if you're signed in, otherwise a git-ignored .env.local).
launchpad secrets run -- cmd
Run a command with your secrets injected in memory — nothing written to disk.

Keep it fresh / fix it

~/Developer/mac-launchpad
Where your setup lives.
launchpad update
Update all your tools (same as bash scripts/update.sh).
launchpad doctor
Health-check your setup; shows green ✔ / red ✘ (same as bash lib/doctor.sh).
npx skills list
See the skills your assistants share (design, documents, browser, workflow).
claude plugin list
See Claude Code's plugins, including the Superpowers workflow.
“something broke — fix it”
Paste any red error text to an assistant and let it fix it.
🪵

If you ever need to send your setup log to someone helping you, it's the file ~/launchpad-setup.log.
